# AIcleaner

AIcleaner 是一个 AI 驱动的文件归类与整理顾问桌面应用。

当前用户入口已经收敛为单一的 `顾问` 页。顾问页内置前置归类流程：你可以在同一个页面选择目录、运行前置归类、查看归类状态，并继续通过顾问对话生成整理计划、预览和执行结果。

## 适合什么场景

- 下载目录、桌面、项目资料盘越来越乱，但你不想靠手工一个个翻。
- 你想先把目录结构理顺，再决定哪些内容该归档、重组或清理。
- 你希望 AI 参与整理，但不希望它一上来就直接批量删除。
- 你有一批混杂的文档、图片、压缩包、素材或项目文件，想先做结构化整理。

## 核心工作流

### 顾问页

顾问页负责完整用户工作流：

- 选择需要处理的工作目录
- 运行内置前置归类流程
- 选择摘要策略和联网搜索开关
- 查看归类任务状态、进度和分类树
- 基于归类结果开启或重建顾问会话
- 用自然语言继续说明整理意图
- 在意图明确后生成计划、预览和执行卡片

归类完成后，用户不在独立页面里直接应用或回滚归类结果。真实文件移动应通过顾问生成的计划、预览和执行卡片完成。

## 第一次使用前需要准备什么

首次使用前，先在左侧边栏点击 `服务商 API`，至少配置一个可用的大模型服务商。

当前界面内已经预置了多种常见服务商入口，例如：

- OpenAI
- DeepSeek
- Gemini
- Qwen
- GLM
- Kimi
- MiniMax

如果你希望前置归类阶段启用联网补充信息，还可以额外配置 Tavily；这不是必选项，没有它也可以完成本地归类和顾问流程。

## 快速开始

### 1. 配置模型服务

点击左侧边栏的 `服务商 API`，填入至少一个可用的 API Key，并选择对应模型。

建议先只配一个你最常用的服务商，确认流程跑通后再扩展。

### 2. 进入顾问页

应用当前只有 `顾问` 入口：

- 选择要处理的目录
- 选择摘要策略
- 视情况打开联网搜索
- 启动前置归类，或直接开始会话

如果当前目录已有最近一次可复用的归类结果，顾问会优先复用；没有时会退回到有边界的目录元信息上下文，不要求你必须先跑归类。

### 3. 需要时迁移缓存

如果你不希望应用数据继续放在当前默认位置，可以使用左侧边栏里的 `迁移缓存`。

这个入口直接嵌在工作流侧边栏里，不需要单独进入设置页。

## 典型使用方式

### 下载目录整理

在顾问页选择下载目录，先运行前置归类，把压缩包、安装包、图片、文档等内容建立成基础分类树；再继续告诉顾问哪些内容该归档、哪些值得继续整理。

### 项目资料归档

对一个项目资料目录运行前置归类，再用顾问页讨论“哪些内容保留在主目录、哪些适合归档、哪些应分流到子目录”。

### 大目录先分后聊

当你不确定一个目录里到底应该怎么整理时，先跑一次前置归类建立基础结构，再让顾问围绕这次结果持续推进。

## 用户能直接感知到的能力

- 单一顾问入口承载前置归类、建议、预览和执行
- 顾问会话支持围绕同一目录持续推进
- 顾问可复用最近一次归类结果
- 真实移动通过计划、预览、执行卡片完成
- 界面支持中英文切换
- 缓存目录可以从侧边栏直接迁移

## 注意事项

- AIcleaner 当前仍处于早期迭代阶段，还不是一个面向生产环境的成熟产品。
- 它更适合“先理解、再整理”的工作流，不适合把它当成无脑批量删除工具。
- 某些能力依赖你配置的模型服务；没有可用 API 时，相关 AI 流程无法正常工作。
- 联网搜索属于可选能力，只有在你配置了对应搜索服务后才可用。
- 当前仓库内容和启动方式都是桌面端形态，不是可直接部署的 Web 应用。

## 最小开发入口

如果你只是想本地跑起来体验当前版本，可使用源码启动。

### 环境前提

- Node.js
- Rust
- Tauri 开发环境

### 启动方式

```bash
npm install
npm run tauri:dev
```

如需本地打包：

```bash
npm run tauri:build
```

当前仓库的脚本和打包配置主要面向桌面端工作流；如果你只是普通用户，阅读上面的产品说明和快速开始部分即可。

# AIcleaner

AIcleaner 是一个 AI 驱动的文件归类与整理顾问桌面应用。

它现在的主工作流已经收敛成两段：`归类 -> 顾问`。你可以直接选择目录开始归类，也可以直接进入顾问页基于目录和已有归类结果继续讨论整理方案。

## 适合什么场景

- 下载目录、桌面、项目资料盘越来越乱，但你不想靠手工一个个翻。
- 你想先把目录结构理顺，再决定哪些内容该归档、重组或清理。
- 你希望 AI 参与整理，但不希望它一上来就直接批量删除。
- 你有一批混杂的文档、图片、压缩包、素材或项目文件，想先做结构化整理。

## 核心工作流

### 1. 归类

归类页负责选择目录，并把文件按主题或用途组织成更清晰的结构。

你可以在这里：

- 选择需要处理的目录
- 配置模型、批大小、摘要策略、排除项等参数
- 查看归类结果、执行日志和预览
- 应用归类动作，并在需要时回滚最近一次结果

### 2. 顾问

顾问页适合“目录很乱，但我还没想好怎么整理”的场景。

它会优先复用最近一次归类结果；如果当前目录还没有可复用的归类记录，也可以直接基于目录元信息启动会话。

你可以在这里：

- 先看整体结构
- 获取自然语言整理建议
- 根据补充要求继续细化方案
- 在更明确的意图下生成可执行预览

## 第一次使用前需要准备什么

首次使用前，先在左侧边栏点击 `服务商 API`，至少配置一个可用的大模型服务商。

当前界面内已经预置了多种常见服务商入口，例如：

- OpenAI
- DeepSeek
- Gemini
- Qwen
- GLM
- Kimi
- MiniMax

如果你希望归类阶段启用联网补充信息，还可以额外配置 Tavily；这不是必选项，没有它也可以完成本地归类和顾问流程。

## 快速开始

### 1. 配置模型服务

点击左侧边栏的 `服务商 API`，填入至少一个可用的 API Key，并选择对应模型。

建议先只配一个你最常用的服务商，确认流程跑通后再扩展。

### 2. 从归类开始

进入 `归类` 页面：

- 选择要处理的目录
- 视情况调整摘要策略、批大小、排除项和联网搜索
- 启动归类

归类完成后，你可以直接应用结果，或者把这次归类结果带入顾问继续讨论。

### 3. 直接进入顾问

进入 `顾问` 页面：

- 选择要处理的目录
- 选择顾问模式
- 启动会话

如果当前目录已有最近一次归类结果，顾问会优先复用；没有时会退回到目录元信息上下文，不要求你必须先跑归类。

### 4. 需要时迁移缓存

如果你不希望应用数据继续放在当前默认位置，可以使用左侧边栏里的 `迁移缓存`。

这个入口直接嵌在工作流侧边栏里，不需要单独进入设置页。

## 典型使用方式

### 下载目录整理

先在归类页处理整个下载目录，把压缩包、安装包、图片、文档等内容拆成更清晰的分类结构；再在顾问页讨论哪些内容该归档、哪些值得继续整理。

### 项目资料归档

先对一个项目资料目录做归类，再用顾问页讨论“哪些内容保留在主目录、哪些适合归档、哪些应分流到子目录”。

### 大目录先分后聊

当你不确定一个目录里到底应该怎么整理时，先跑一次归类建立基础结构，再让顾问围绕这次结果持续推进。

## 用户能直接感知到的能力

- 归类结果支持执行与回滚
- 顾问会话支持围绕同一目录持续推进
- 顾问可复用最近一次归类结果
- 界面支持中英文切换
- 缓存目录可以从侧边栏直接迁移

## 注意事项

- AIcleaner 当前仍处于早期迭代阶段，还不是一个面向生产环境的成熟产品。
- 它更适合“先理解、再整理”的工作流，不适合把它当成无脑批量删除工具。
- 某些能力依赖你配置的模型服务；没有可用 API 时，相关 AI 流程无法正常工作。
- 联网搜索属于可选能力，只有在你配置了对应搜索服务后才可用。
- 当前仓库内容和启动方式都是桌面端形态，不是可直接部署的 Web 应用。

## 最小开发入口

如果你只是想本地跑起来体验当前版本，可使用源码启动。

### 环境前提

- Node.js
- Rust
- Tauri 开发环境

### 启动方式

```bash
npm install
npm run tauri:dev
```

如需本地打包：

```bash
npm run tauri:build
```

当前仓库的脚本和打包配置主要面向桌面端工作流；如果你只是普通用户，阅读上面的产品说明和快速开始部分即可。
